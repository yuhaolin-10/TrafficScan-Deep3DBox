# TrafficScan UI 方案 A

## 1. 文档目的

本文档用于沉淀 TrafficScan 当前阶段的精简版 UI 改版方案 A。

方案 A 的目标不是重做产品结构，而是在保留现有桌面端三栏框架的前提下，把界面从“工程工具台”整理为“新手第一次打开就知道怎么用”的任务流界面。

本文档重点覆盖：

- 页面信息架构
- 各区域显示内容
- 空状态与运行状态
- ROI 区域交互
- 区域规则展示方式
- 当前阶段需要隐藏或延后的功能

## 2. 设计目标

### 2.1 目标

- 第一次打开时，用户能在 5 秒内理解主流程
- 导入、框选、分析、查看结果这 4 个动作足够清晰
- 运行过程中有明确反馈，不让用户怀疑软件卡住
- 右侧信息区只展示当前真正需要的信息
- 高级显示能力先从前台隐藏，但后端能力保留

### 2.2 非目标

- 本轮不做完整产品级多页面重构
- 本轮不把图层切换做成主界面能力
- 本轮不恢复车道分割相关展示
- 本轮不做复杂筛选、统计报表或历史分析大屏

## 3. 当前阶段的产品原则

### 3.1 主流程优先

优先服务这条路径：

1. 从左侧导入素材
2. 需要时框选区域
3. 给区域设置规则
4. 点击开始分析
5. 在中间查看结果，在右侧查看选中区域规则

### 3.2 少即是多

界面上只保留会直接影响主流程的控件。

不把以下能力放到第一层：

- 图层开关
- 技术参数展示
- 调试日志
- 复杂筛选
- 车道相关显示

### 3.3 信息分层

界面上出现的信息要满足：

- 顶部只负责主操作和运行反馈
- 左侧只负责素材导入与切换
- 中间只负责预览和区域交互
- 右侧只负责当前选中区域的规则信息

## 4. 页面结构总览

### 4.1 主界面结构

主界面继续采用三栏布局：

- 左侧：工作区
- 中间：预览与区域交互区
- 右侧：选中区域规则

### 4.2 结构草图

```text
+--------------------------------------------------------------------------------------+
| TrafficScan                                   [开始分析]                             |
|                                             运行中时显示：进度条 + 当前处理提示       |
+----------------------+------------------------------------------------+-------------+
| 工作区               | 预览区                                         | 选中区域规则 |
|                      |                                                |             |
| [导入卡片]           | [ROI 按钮条]                                   | 未选中时提示 |
|                      |                                                | 或           |
| [素材列表]           | [空状态说明 / 图片 / 视频首帧 / 分析结果]       | 当前区域规则 |
|                      |                                                |             |
| [清空] [移除]        | [图片模式无播放控件 / 视频模式显示播放控件]     |             |
+----------------------+------------------------------------------------+-------------+
```

## 5. 顶部区域设计

## 5.1 默认状态

顶部只保留以下元素：

- 产品名：`TrafficScan`
- 主按钮：`开始分析`

默认状态下不长期显示状态文字，不显示“空闲”“已完成”这类常驻提示。

原因：

- 顶部区域应保持干净
- 不与右侧内容区产生“状态信息重复”
- 用户主要注意力应留给导入和分析动作

## 5.2 运行中状态

只有在运行中时，顶部才额外出现运行反馈。

建议表现方式：

- 一条横向细进度条
- 一句运行提示
- 一个停止按钮

建议文案：

- `正在分析 3/20 · illegal_2.mp4`
- `正在分析 1/8 · 365A2050.jpg`

## 5.3 运行结束反馈

运行结束后显示一次短暂反馈，然后恢复默认简洁状态。

建议文案：

- `分析完成，共发现 4 条疑似违规`
- `分析完成，未发现疑似违规`
- `分析已停止`

## 6. 左侧工作区设计

## 6.1 区域职责

左侧只负责两件事：

- 导入素材
- 切换当前查看的素材

本轮不加入筛选器，不加入额外统计面板。

## 6.2 导入卡片

左上方保留现有导入区，但文案更偏任务导向。

建议文案：

- 标题：`导入图片或视频`
- 副文案：`拖拽到这里，或点击导入文件 / 文件夹`

导入区应明确说明支持：

- 图片
- 视频
- 文件夹

## 6.3 素材列表

每个列表项建议显示：

- 缩略图
- 文件名
- 类型标记：图片 / 视频
- 状态标记：待分析 / 分析中 / 已完成 / 失败
- 简短结果摘要

结果摘要建议规则：

- 未分析：`未分析`
- 已完成且有结果：`2 条疑似违规`
- 已完成且无结果：`未发现违规`
- 分析失败：`分析失败`

## 6.4 列表风格建议

不再使用当前这种：

```text
[pending] [image] xxx.jpg
```

建议改成更接近产品卡片式列表，让非技术用户一眼能看懂状态。

## 6.5 底部按钮

保留：

- `清空`
- `移除`

不新增其他批量筛选控件。

## 7. 中间预览区设计

## 7.1 区域职责

中间区域承担 3 个职责：

- 显示图片或视频预览
- 完成 ROI 框选
- 在分析后查看叠加结果

本轮不把图层切换放在主界面第一层。

## 7.2 ROI 按钮条

中间上方只保留和区域交互直接相关的按钮：

- `开始框选`
- `完成框选`
- `清空区域`
- `保存规则`

说明：

- `保存规则` 的实际含义是保存当前场景下的区域与规则配置
- 本轮先用这个文案，后续如果需要更准确，可再调整为 `保存当前配置`

## 7.3 隐藏的图层能力

当前不在前台显示：

- 区域覆盖层开关
- 底面开关
- 3D 框开关
- 文本标注开关

但后端应保留这些能力与状态接口，为未来版本的“高级显示”抽屉做准备。

### 7.3.1 后续建议

未来若恢复显示控制，建议不要做成 4 个平铺按钮，而是放到：

- `高级显示`

展开后再提供：

- 区域
- 底面
- 3D 框
- 文字标注

## 7.4 图片与视频的轻区分

本轮不拆成两个页面，但在中间区域做轻区分：

- 图片素材：只显示预览
- 视频素材：显示预览 + 播放条 + 跳帧控件

建议在预览区左上角显示小标签：

- `图片分析`
- `视频分析`

## 8. 中间区域的空状态设计

## 8.1 设计原则

完全未导入时，中间区域不再重复提供导入按钮。

原因：

- 左侧已经有明确导入入口
- 中间再放导入入口会造成功能重复
- 中间区域更适合承担“说明”和“预期结果提示”

## 8.2 完全未导入时

中间区域显示“说明型空状态”。

建议结构：

- 一张淡化的示意图或占位插画
- 主文案：`左侧导入图片或视频后，这里会显示预览与分析结果`
- 3 步说明

建议文案：

1. `从左侧导入素材`
2. `需要时框选区域并设置规则`
3. `点击开始分析`

底部补充一行小提示：

- `分析完成后，可在这里点击区域查看结果`

## 8.3 已导入但未分析时

中间区域显示：

- 当前原图或视频首帧
- 顶部轻提示条

建议提示文案：

- `素材已导入，下一步点击“开始分析”`

## 8.4 已分析时

中间区域显示分析结果叠加图。

如果后端已经生成结果，就用结果图替换原始预览。

## 9. ROI 交互设计

## 9.1 主交互方式

本轮不依赖右键去发现核心能力。

主交互应通过固定按钮完成。

## 9.2 建议流程

1. 用户点击 `开始框选`
2. 中间区域进入框选模式
3. 顶部出现持续提示：`左键依次点击，双击完成当前区域`
4. 完成后自动退出本次绘制
5. 右侧区域更新为当前选中区域的规则信息

## 9.3 框选中的提示

建议统一用显性提示，不要求用户记忆复杂规则。

建议文案：

- 开始后：`左键依次点击，双击完成当前区域`
- 完成后：`已完成 1 个区域，可继续框选或保存规则`
- 未满足条件时：`当前区域至少需要 3 个点`

## 9.4 清空逻辑

点击 `清空区域` 后：

- 清空当前场景的所有已绘制区域
- 右侧规则面板恢复未选中状态
- 如果已有历史分析结果，应提示这些结果可能失效

## 9.5 保存逻辑

点击 `保存规则` 后：

- 保存当前场景下所有已绘制区域
- 保存每个区域启用的规则
- 下次进入同一场景时自动恢复

## 10. 右侧规则面板设计

## 10.1 区域职责

右侧本轮只做一件事：

- 展示当前选中区域已启用的规则

不再承担：

- 文件路径
- 图片尺寸
- 运行统计
- 技术调试信息

## 10.2 标题建议

建议标题直接使用：

- `选中区域规则`

避免使用：

- 详情
- 场景设置
- 分析状态

这些词都不够聚焦，容易让用户不知道右侧到底看什么。

## 10.3 未选中区域时

右侧显示提示态。

建议文案：

- `点击中间画面中的某个区域查看规则`

如果当前一个区域都没有，可以改成：

- `当前还没有区域，先点击“开始框选”创建区域`

## 10.4 选中区域时

右侧建议显示以下信息：

- 区域名称
- 是否启用
- 已启用规则列表

规则列表建议只显示启用项：

- `禁止停车`
- `禁止非机动车`
- `禁止逆行`

若启用了 `禁止逆行`，额外显示：

- `允许方向：已设置`
或
- `允许方向：未设置`

## 10.5 右侧交互建议

如果本轮要把规则启停也做到显性化，建议在右侧使用开关项，而不是要求用户通过右键菜单发现。

建议形式：

- `禁止停车` 开关
- `禁止非机动车` 开关
- `禁止逆行` 开关

当 `禁止逆行` 打开后，额外出现：

- `设置允许方向`

## 11. 运行反馈设计

## 11.1 为什么必须做

当前桌面端最容易影响信任感的问题，是点击运行后用户不确定系统是否仍在工作。

因此运行反馈是本轮最高优先级之一。

## 11.2 运行反馈最低要求

运行中至少要让用户看到：

- 当前正在处理第几个
- 总共有多少个
- 当前文件名
- 是否还能停止

## 11.3 建议表现

建议在顶部显示：

- 细进度条
- 文案：`正在分析 3/20 · illegal_2.mp4`
- 按钮：`停止`

## 11.4 完成后的反馈

完成后短暂展示结果提示：

- `分析完成，共发现 4 条疑似违规`
- `分析完成，未发现疑似违规`

## 12. 本轮延后显示的内容

以下内容后端保留，但本轮不在主界面第一层显示：

- 图层开关
- 底面 / 3D 框切换
- 文字标注开关
- 技术详情长文本
- 调试日志
- 复杂筛选
- 车道分割相关展示

这些能力建议在未来版本中放入：

- `高级显示`
- `技术详情`
- `调试信息`

而不是直接堆在主界面上。

## 14. 实施优先级

## 14.1 P0

- 顶部运行进度反馈
- 左侧列表样式重构
- 中间空状态重做
- 右侧改成“选中区域规则”
- ROI 固定按钮交互

## 14.2 P1

- 右侧规则开关显性化
- 视频与图片模式标签
- 保存规则后的反馈提示

## 14.3 P2

- 高级显示抽屉
- 技术详情折叠区
- 调试日志入口

## 15. 验收标准

以下标准满足时，可视为方案 A 的第一轮改版达标：

1. 新用户第一次打开时，不需要阅读文档也知道先从左侧导入素材
2. 用户导入素材后，能明确知道下一步是开始分析
3. 用户在分析运行中能明确看到进度，而不是怀疑程序卡住
4. 用户框选区域后，能在右侧直接看到该区域启用的规则
5. 主界面不再出现大段技术文本或过多高级按钮
6. 车道分割移除后，界面文案和结构仍然自洽

## 16. 结论

方案 A 的核心不是增加功能，而是做减法。

当前阶段应把 UI 收敛为：

- 左侧导入与切换
- 中间预览与框选
- 右侧区域规则
- 顶部运行反馈

只要先把这 4 件事做清楚，第一次上手的可理解性就会明显提升，同时也为后续再开放高级显示能力留出了空间。
