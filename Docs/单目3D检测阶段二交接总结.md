# 单目3D检测阶段二交接总结

## 1. 项目背景与目标

当前项目是固定高速场景下的单目 3D 车辆检测，用于辅助应急车道占用识别。

当前用户最关心的两个问题是：

1. 3D box 朝向经常整体偏掉，不够顺着道路。
2. 深度和方向容易受单目估计误差影响，导致框太大、朝向不稳定。

阶段二的目标是：

1. 把车道先验真正接入 yaw。
2. 优先通过几何和场景先验提高稳定性，而不是马上重训大模型。

---

## 2. 当前代码已经完成的基础能力

### 2.1 阶段一几何修正已经完成

已经完成的内容包括：

1. 对深度采样做了更稳健的 mask 内取样。
2. 深度 hint 与几何 depth prior 做了保守融合。
3. translation 做了小范围搜索和重投影评分。
4. 3D 结果增加了 `debug_3d` 调试信息。

主要文件：

1. `src/core/vehicle_detector_3d.py`
2. `src/services/pipeline.py`
3. `src/tests/test_vehicle_detector_3d_geometry.py`

### 2.2 vehicle mask 可视化已经完成

已经可以在 GUI 中显示车辆实例分割 mask，并支持开关图层。

### 2.3 局部 lane direction 已接入 yaw，但目前是软约束

当前 `vehicle_detector_3d.py` 里已经支持：

1. 从 `lane_geometry.py` 获取每辆车对应的局部车道方向。
2. 将 lane prior 作为 soft blend 接入 yaw。
3. 在正负 yaw 候选里结合重投影误差做最终选择。

重要事实：

1. 现在 lane prior 不是硬替换 yaw。
2. 现在 lane prior 只是“拉一把”，不是“直接决定 3D 框方向”。

## 3. 阶段二中已经尝试过的方向

## 3.1 最早版本：原始 polygon 点云 PCA

做法：

1. 对整块 lane polygon 的点做 PCA 得主方向。
2. 对车辆附近取最近一批 polygon 点，再做局部 PCA 得局部箭头。

结果：

1. 全局箭头明显不可靠，经常指错。
2. 局部箭头在 lane 分割质量好的图上有用。
3. 一旦车道边界有缺口、鼓包、锯齿，局部箭头容易被带偏。

结论：

1. 全局箭头不适合参与决策。
2. 只能保留局部方向思路。

---

## 3.2 第二版：边界提取 + 平滑 + 跨缺口合并

做法：

1. 从 lane 区域逐行提左右边界。
2. 对边界做重采样和平滑。
3. 去掉边界顶端和底端一部分再拟合。
4. 允许同侧上下断裂边界在条件满足时合并。
5. 加入“单侧优先边界选择”，即如果某一侧明显更平滑，就给该侧加分。

当前保留的代码状态就是这一版。

这一版涉及的核心文件：

1. `src/core/lane_segmenter.py`
2. `src/core/lane_geometry.py`

效果评价：

1. 比原始 polygon PCA 更稳定。
2. 对小缺口、小锯齿有一定改善。
3. 对一侧明显更平滑的图，有时能起正作用。
4. 仍然会受到同一块车道区域上下部分形状的影响。
5. 某些车会被错误边界趋势带偏。

结论：

1. 这是目前“还能用的较好版本”。
2. 可以作为继续试验的起点。

---

## 3.3 第三版：局部纵向 ROI + 更强局部单侧约束

做法：

1. 对每辆车只看锚点附近一小段纵向边界。
2. 把单侧优先从全局偏好改成局部偏好。
3. 尽量减少上下远处边界对当前车辆的影响。

结果：

1. 少数问题车有改善。
2. 但整体稳定性变差，用户反馈“全部乱套了”。
3. 局部约束过强后，整体方向一致性反而更差。

当前状态：

1. 这一个版本已经明确回退。
2. 当前代码不再包含这版的局部纵向 ROI 强约束。

结论：

1. 这条路线短期内不建议继续直接推进。
2. 如果以后再试，必须加更强的可视化和更温和的约束方式。

---

## 5. 当前用户观察到的核心问题

用户实际测试后的反馈可以概括为：

1. 应急车道分割质量好的图，局部箭头和 3D 朝向确实有帮助。
2. 但不少图里，只要某一段边界质量差，仍会把个别车的方向带偏。
3. 典型问题是：某辆车被上下区域或错误边界趋势影响，箭头明显不合理。
4. 用户怀疑：
   - 是不是应当排除上下区域的影响
   - 是不是应当只关注某一侧更平滑的边界

其中第二点“优先更平滑的一侧”已经尝试过温和版，并保留在当前代码里。

## 6. 当前最值得继续尝试的方向

## 6.1 重新评估“全局方向”方案，但方式要升级

用户最新反馈是：有点想重新启用并优化最开始那个全局方向方案。

这里不建议回到最原始的“整块 polygon PCA 全局箭头”，而建议考虑下面这个更稳的版本：

1. 不用整块区域散点主轴。
2. 改为基于更可靠那一侧边界，拟合一条全局主方向。
3. 这个全局方向只用作 base yaw 或稳定先验。
4. 每辆车的局部方向只做 residual 或小修正。

也就是从：

`最终方向 = 车辆视觉 yaw + 局部 lane prior`

改成更像：

`最终方向 = 全局/主边界 base direction + 局部微调 + 视觉 residual`

这条路很可能比“极端局部化”更稳。

## 6.2 值得继续的具体候选方案

推荐下一轮重点比较以下三种方案，而不是继续在一个方案上盲调。

### 方案 A：全局主方向回归

做法：

1. 只用较优那一侧边界。
2. 拟合整段主方向。
3. 让所有车辆共享一个基础方向。

优点：

1. 稳定。
2. 同车道车更一致。

缺点：

1. 对局部弯曲适应较弱。

### 方案 B：全局 base + 局部 residual

做法：

1. 全局主方向提供 base yaw。
2. 每辆车局部边界只提供一个小修正。

优点：

1. 比纯全局更灵活。
2. 比纯局部更稳。

缺点：

1. 需要重新设计融合逻辑。

### 方案 C：按车道区域中心线建模

做法：

1. 左右边界都提出来。
2. 先求中线。
3. 用中线切线作为方向。

优点：

1. 语义更自然。
2. 不会过于依赖某一侧轮廓。

缺点：

1. 当一侧边界很差时，中线也会被拖累。

---

## 7. 当前不建议立即继续的方向

1. 不建议继续直接强化“每辆车只看很窄的局部纵向范围”。
   原因：上一轮已经证明整体容易乱。

2. 不建议现在马上重训 lane segmentation 模型。
   原因：当前主要瓶颈仍然是“如何从 lane 结果里提稳定方向”。

3. 不建议继续使用原始全局 polygon PCA 箭头。
   原因：这个全局箭头已经证明会严重误导。

6. 下一步希望重新评估“全局方向”方案，但不是回到原始 polygon PCA，而是考虑：
   - 基于优质单侧边界的全局主方向
   - 或“全局 base + 局部 residual”的融合方案

## 9. 建议新对话优先回答的问题

建议让新的对话优先聚焦这几个问题：

1. 在当前高速固定相机场景下，方向估计更适合“全局 base + 局部修正”还是“纯局部切线”？
2. 如果重新引入全局方向，最稳的全局方向提取方式是什么？
3. 单侧优先是否应升级为“只用优质侧边界求全局主方向”？
4. 如何把 lane prior 从当前 soft blend，升级成更稳定但不过强的 yaw 约束？

---

## 10. 一句话结论

当前最值得继续的路线不是再把局部逻辑越做越窄，而是：

**在当前保留版本基础上，重新设计一个更稳定的全局主方向，再让局部信息只做修正。**
