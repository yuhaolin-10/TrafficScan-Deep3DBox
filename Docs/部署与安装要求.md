# TrafficScan Deep3DBox UI 部署与安装要求

本文档用于在新机器上复现本项目的运行环境。项目主体是基于 Python + Qt 的桌面端应用，核心能力包括车道分割、车辆检测、Deep3DBox 单目 3D 估计、违章判定、结果渲染与 SQLite 数据持久化。`Figma/Traffic Violation Detection App` 目录是独立的 React/Vite 前端原型，不影响 Python 桌面程序运行。

## 1. 推荐运行环境

- 操作系统：Windows 10/11 64 位。
- Python：推荐 Python 3.10。不要优先使用 Python 3.11/3.12/3.13，因为 Deep3DBox 依赖的 TensorFlow 2.10.x 在 Windows 原生环境下对 Python 3.10 更稳定。
- 包管理器：推荐 Anaconda/Miniconda，也可以使用 `venv`。
- 硬件：
  - CPU 可运行完整流程，但模型推理速度较慢。
  - 推荐 NVIDIA GPU，显存建议 6 GB 以上。
  - 若使用 GPU，需要提前安装匹配 PyTorch/TensorFlow 的 CUDA/cuDNN 环境；没有 GPU 时程序会回退到 CPU。
- 磁盘空间：建议至少预留 5 GB。模型权重、处理后图片、视频结果和数据库会占用额外空间。

## 2. 项目目录要求

复现时应保持以下关键文件和目录存在：

```text
TrafficScan_Deep3DBox_UI/
  src/
    run_gui_auto_batch.py
    main_system.py
    main_image_test.py
    core/
    gui/
    services/
    models/
      best.pt
      yolo11l.pt
      yolo11l-seg.pt
  external/
    deep3dbox_demo_model/
      demo_model.data-00000-of-00001
      demo_model.index
      demo_model.meta
  images/
  data/
  requirements.txt
```

其中必须具备的模型文件为：

- `src/models/best.pt`：车道分割模型。
- `src/models/yolo11l.pt`：车辆 2D 检测模型。
- `external/deep3dbox_demo_model/demo_model.*`：Deep3DBox checkpoint，代码默认读取不带后缀的路径 `external/deep3dbox_demo_model/demo_model`。

如果这些权重文件没有随项目一起分发，需要从原训练/实验环境复制到对应位置，否则 GUI 或批处理流程会在模型初始化阶段报错。

## 3. 创建 Python 环境

推荐使用 conda：

```powershell
conda create -n trafficscan-deep3dbox python=3.10 -y
conda activate trafficscan-deep3dbox
```

进入项目根目录：

```powershell
cd D:\path\to\TrafficScan_Deep3DBox_UI
```

安装依赖：

```powershell
python -m pip install --upgrade pip
pip install -r requirements.txt
```

说明：

- `PySide6` 用于桌面 GUI。代码也保留了 PyQt5 回退逻辑，但推荐直接安装 PySide6。
- `ultralytics` 用于加载 YOLO 权重。
- `tensorflow==2.10.1` 和 `tf_slim` 用于 Deep3DBox 的 TensorFlow v1 兼容推理。
- `torch` 用于 YOLO 推理和可选深度估计。若需要特定 CUDA 版本的 PyTorch，请按 PyTorch 官方命令单独安装对应 wheel，然后再安装其余依赖。
- `transformers` 仅用于可选的 ZoeDepth 深度估计模块；即使该模块初始化失败，主流程会回退到原有深度先验逻辑。

## 4. 运行前检查

在项目根目录执行以下命令，检查关键文件是否存在：

```powershell
Test-Path src\models\best.pt
Test-Path src\models\yolo11l.pt
Test-Path external\deep3dbox_demo_model\demo_model.index
```

三个命令都应输出 `True`。

可选：检查 Python 能否导入关键依赖：

```powershell
python -c "import cv2, numpy, torch, tensorflow, tf_slim; from PySide6 import QtWidgets; from ultralytics import YOLO; print('OK')"
```

## 5. 启动桌面 GUI

项目根目录下运行：

```powershell
python src\run_gui_auto_batch.py
```

也可以直接双击或执行：

```powershell
.\run_gui_auto_batch.bat
```

该入口会启动 Qt 主窗口，并尝试自动导入以下目录中的测试图片：

```text
images/test_images/highway
images/test_images/illegal
```

如果本地没有这两个目录，程序会在日志中提示缺失。仍可在 GUI 中手动导入图片或视频素材。

## 6. 批处理与验证命令

单图/图片目录集成测试：

```powershell
python src\main_image_test.py
```

该脚本默认读取：

```text
images/test_images/illegal
```

输出结果默认保存到：

```text
images/integrated_test_results
```

带数据库持久化的系统流程：

```powershell
python src\main_system.py
```

输出目录和数据库默认位置：

```text
data/images
data/db/traffic_scan.db
```

Deep3DBox 独立验证：

```powershell
python src\tools\run_deep3dbox_illegal_test.py
```

该脚本默认使用：

```text
src/models/yolo11l.pt
external/deep3dbox_demo_model/demo_model
```

## 7. 测试

运行单元测试：

```powershell
pytest src\tests
```

注意：部分测试只验证几何、规则、数据库等纯逻辑；涉及真实模型推理的功能需要模型文件和完整依赖环境。

## 8. 数据与输出目录

项目运行时会使用或生成以下目录：

- `images/`：输入图片、视频和演示素材。
- `data/images/`：处理后的图片结果。
- `data/videos/`：处理后的视频结果。
- `data/video_previews/`：视频预览图。
- `data/db/traffic_scan.db`：SQLite 结果数据库。
- `data/scene_profiles/`：场景配置、区域规则等 JSON 文件。

复现实验时建议保留 `src/models/`、`external/deep3dbox_demo_model/` 和必要的 `images/` 输入数据；`data/` 下的运行结果可以按需清空后重新生成。

## 9. 可选功能

### 9.1 车牌识别实验

车牌识别 POC 位于：

```text
experiments/license_plate_poc
```

它有单独依赖：

```powershell
pip install -r experiments\license_plate_poc\requirements.txt
```

当前 `core/plate_recognizer.py` 会尝试使用 HyperLPR，但如果缺少 `experiments/license_plate_poc/vendor` 或相关依赖，会自动退化为禁用车牌识别，不影响主检测流程。

### 9.2 React/Vite 前端原型

前端原型目录：

```text
Figma/Traffic Violation Detection App
```

运行方式：

```powershell
cd "Figma\Traffic Violation Detection App"
npm install
npm run dev
```

构建：

```powershell
npm run build
```

该前端原型与 Python 桌面端不是同一个运行入口，复现核心算法和 GUI 时无需启动它。

## 10. 常见问题

### 10.1 找不到模型文件

报错类似：

```text
YOLO weights not found
Deep3DBox checkpoint not found
Lane segmentation model not found
```

处理方法：确认以下路径存在：

```text
src/models/best.pt
src/models/yolo11l.pt
external/deep3dbox_demo_model/demo_model.index
```

### 10.2 TensorFlow 安装失败

优先确认 Python 版本是否为 3.10。Windows 原生环境下不建议用 Python 3.11 及以上安装本项目的 Deep3DBox 环境。

### 10.3 Qt 插件或窗口启动异常

优先使用 conda 环境重新安装 PySide6：

```powershell
pip uninstall PySide6 PyQt5 -y
pip install PySide6
```

同一个环境里尽量不要混装多套 Qt 绑定。

### 10.4 GPU 显存不足

可先使用 CPU 运行验证，或降低输入分辨率、减少批处理数量。代码中 YOLO 与 TensorFlow 会共享 GPU，`VehicleDetector3D` 已在 Deep3DBox 推理前尝试释放 PyTorch GPU 缓存，但显存较小时仍可能失败。

## 11. 最小复现流程

在一台新 Windows 机器上，最小复现步骤如下：

```powershell
conda create -n trafficscan-deep3dbox python=3.10 -y
conda activate trafficscan-deep3dbox
cd D:\path\to\TrafficScan_Deep3DBox_UI
pip install -r requirements.txt
python -c "import cv2, torch, tensorflow, tf_slim; from PySide6 import QtWidgets; from ultralytics import YOLO; print('OK')"
python src\run_gui_auto_batch.py
```

若 GUI 能正常打开、模型能完成初始化，并能对导入的图片或视频生成检测结果，即说明部署完成。
