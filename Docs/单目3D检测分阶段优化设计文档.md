# TrafficScan 单目3D检测分阶段优化设计文档

## 一、文档目的

本文档用于整理 TrafficScan 项目中单目 3D 检测模块的后续优化路线，重点围绕当前系统在实际效果中暴露出的两个主要问题展开：

1. 预测得到的 3D box 普遍偏大，表现为框体向外扩张、底面过宽、远处车辆被估计得过近。
2. 车辆朝向估计整体存在系统性偏差，部分车辆虽然大致方向正确，但 yaw 偏移较明显，影响 3D 框的透视效果和几何稳定性。

当前项目并不是一个端到端的标准单目 3D 检测网络，而是一个“检测/分割结果 + 单目深度图 + 几何推理”的工程型方案。其核心实现位于 [`src/core/vehicle_detector_3d.py`](d:\college\GraduationProject\testProject\TrafficScan_lab_copy\src\core\vehicle_detector_3d.py) 中，整体流程可以概括为：

1. 用 YOLO 得到车辆 2D 检测框和可选实例分割轮廓。
2. 用深度估计模块输出整张图的 depth map。
3. 根据 2D 框尺寸、车辆尺寸先验、深度采样结果和启发式朝向规则恢复 3D box。
4. 将 3D box 的底面投影作为车辆占道区域，用于后续违章判定。

因此，本文档的优化路线不以“直接更换为更大的单目 3D 模型”为前提，而是以现有代码基础为出发点，按照“先修几何链路，再引入场景先验，最后考虑轻量学习模块”的思路逐步推进。

## 二、当前方案的问题定位

### 2.1 3D box 偏大的主要原因

从当前实现看，3D box 偏大不是单一因素造成的，而是多个误差叠加的结果。

第一，深度估计结果在当前系统中的权重偏高。代码中 `DEPTH_HINT_BLEND = 0.85`，说明一旦深度图在某辆车上采样到偏近的深度值，最终用于恢复 3D box 的深度就会明显偏向这个值。对单目深度估计而言，底部区域尤其容易被路面、阴影、轮胎和遮挡边缘干扰，因此一旦采样不稳，就会把车辆整体“拉近”。

第二，当前深度采样主要发生在 2D 框底部区域。虽然这一设计本意是优先利用车辆接地部分的信息，但在高速道路场景中，车辆底部恰好也是最容易混入路面区域的地方，因此容易产生“车比实际更近”的错误判断。

第三，相机参数没有经过真实标定，而是通过 `FOCAL_LENGTH_SCALE` 近似估算得到。对于单目 3D 恢复任务，相机焦距误差会直接影响深度、尺寸以及投影结果，因此这是一个系统性误差源。

第四，当前 translation 估计方式较为粗糙，3D box 底面中心是直接锚定到 2D 框底边中心附近，而不是通过更严格的重投影一致性来反推真实 3D 位置。这会导致一旦深度或朝向稍有偏差，整个 3D box 就会被放大或偏移。

### 2.2 朝向估计偏差的主要原因

当前 yaw 恢复主要依赖以下几类启发式信息：

1. 2D 检测框的宽高比。
2. mask 主轴方向。
3. 车辆在画面中相对于图像中心的左右位置。
4. 预设消失点的几何关系。

这种做法的优点是实现简单，不需要额外训练姿态模型；但问题在于它缺乏真正可学习的姿态表征，也没有将“局部朝向”和“全局朝向”区分开来。结果就是：只要车辆在图像中的位置、遮挡情况、远近尺度发生变化，yaw 就容易整体偏移。

尤其需要注意的是，当前规则中存在“按目标位于图像中心左边还是右边来决定符号”的逻辑。这个规则在一些场景下能产生看似合理的结果，但本质上过于强硬，容易造成系统性偏航。

### 2.3 为什么这个问题适合分阶段优化

当前系统已经完成了违章识别主链路，能够输出检测结果、渲染 3D 框并进行入库。因此现阶段更适合做“增量式修复”而不是“推倒重来”。

项目的优势在于：

1. 已有车辆检测结果和实例分割轮廓。
2. 已有单目深度模块接口。
3. 已有车道分割结果。
4. 目标场景较固定，车辆大多位于道路平面上，并沿车道方向运动。

这些条件意味着：即使不立刻上更复杂的大模型，也可以通过更合理的几何约束和场景先验，先把当前方案稳定下来。

## 三、分阶段优化路线

## 阶段一：不换大模型，只修几何链路

### 3.1 阶段目标

这一阶段的核心目标不是追求论文级指标，而是先修复当前 3D box 的明显几何失真，让系统从“能出框”提升到“框体大体可信、朝向不再系统性偏移”。

### 3.2 主要优化点

#### 3.2.1 调整深度融合策略

当前方案中，depth map 的采样结果对最终深度影响过强。第一阶段应先降低深度图的直接支配权，改为“几何先验深度”和“深度图深度”共同决定最终深度。

建议做法：

1. 保留基于车辆尺寸先验和 2D 框大小得到的几何深度。
2. 对 depth map 提供的深度值做过滤和置信度判断。
3. 不再固定使用很高的融合比例，而是根据检测框尺寸、mask 质量、depth map 局部方差等信息动态决定深度图的可信度。

这样做的好处是：即使深度图在某些车辆上采样失真，也不会立刻把最终 3D box 拉崩。

#### 3.2.2 优化 depth 采样区域

当前的 depth 采样重点放在 bbox 的底部条带区域，这一设计虽然直观，但容易采到路面深度。第一阶段应将“如何采样深度”作为一个独立问题处理。

建议做法：

1. 优先只在实例分割 mask 内采样，不使用整块 bbox。
2. 对 mask 最底部若干像素做裁剪，尽量避开轮胎和路面交界区域。
3. 用中位数前，先做分位数过滤，去掉极小值和极大值。
4. 如果局部 depth 的波动过大，则降低它在融合中的权重。

#### 3.2.3 用重投影一致性微调 translation

当前 translation 的求法偏“直接摆放”，而不是“从投影关系反求”。第一阶段不需要引入复杂优化器，但可以用轻量方式增强几何一致性。

建议做法：

1. 在初始深度和 yaw 给出后，生成初始 3D box。
2. 将 3D box 再投影回图像。
3. 检查投影后的底边、左右边界、底面 footprint 是否与原始 2D 检测框和实例分割轮廓对齐。
4. 若偏差较大，则在小范围内搜索更合适的 depth 和水平偏移量。

这一过程的本质是：不再单纯相信初值，而是要求“算出来的 3D 解投回 2D 后也要看起来合理”。

#### 3.2.4 弱化硬编码 yaw 符号规则

当前朝向偏差的一个重要来源，是 yaw 的正负方向带有明显的硬规则色彩。第一阶段应先把“图像中心左右决定方向”的强规则降级为弱参考，而不是最终决定项。

建议做法：

1. 朝向符号不能只由目标中心点与图像中心的相对位置决定。
2. 允许 bbox 形状、mask 主轴和投影一致性共同参与判断。
3. 当多个线索相互冲突时，优先选择使重投影更合理的朝向结果。

#### 3.2.5 校正相机焦距近似值

如果短期内没有正式标定条件，至少应在当前固定机位场景下做一次“经验校正”。

建议做法：

1. 选取几张具有代表性的测试图像。
2. 人工观察哪些车辆的框大小最接近真实透视。
3. 用这些图来校准焦距比例参数，使系统在当前机位下不再普遍出现“近小远大不合理”的情况。

### 3.3 第一阶段预期效果

完成后，系统应至少达到以下效果：

1. 同一张图中，大部分车辆的 3D box 不再普遍偏大。
2. 远处车辆的框不会明显向外发散。
3. 朝向估计不再整批同方向偏转。
4. 3D box 底面与车辆接地区域更贴合。

### 3.4 第一阶段风险

1. 如果深度图本身质量极差，仅靠融合策略很难彻底修复。
2. 如果相机焦距误差过大，仅做经验调整仍可能存在残余偏差。
3. 若 2D 检测框和实例分割本身不稳，重投影微调的收益会受到限制。

## 阶段二：把 lane/ground prior 真正接入 depth 和 yaw

### 4.1 阶段目标

第一阶段主要是在“现有几何链路”内部做修复；第二阶段则进一步利用项目特有的场景先验，也就是车道和地面信息，让系统不再单纯依赖 bbox 和 depth map，而是开始利用道路结构来约束 3D box。

### 4.2 为什么这一阶段非常适合本项目

TrafficScan 不是一个通用自动驾驶 benchmark 项目，而是一个目标场景较固定的高速道路违章检测系统。与通用单目 3D 检测相比，它有两个天然优势：

1. 车辆几乎总是在地面上。
2. 车辆的行驶方向大多与车道方向一致，尤其在高速和应急车道场景中更明显。

这些先验如果不用，就等于浪费了场景优势。

### 4.3 主要优化点

#### 4.3.1 将车道方向作为 yaw 先验

当前 yaw 更多来自 bbox 的外观启发式，而第二阶段建议改为：

1. 从 lane mask 或 lane polygon 中提取局部车道切线方向。
2. 将该方向作为车辆主朝向的先验。
3. 最终车辆朝向不再直接“从零猜测”，而是在车道方向附近做偏差估计。

这样做的原因是：在高速道路场景下，车辆不可能大角度横着停在路上，因此车道方向本身就是非常有力的姿态约束。

#### 4.3.2 将地面平面作为深度约束

车辆底面必须接触地面，因此任何一个合理的 3D box 都应满足：

1. 底面不会漂浮在道路上方。
2. 底面不会插入道路内部。
3. 底面与路面走向基本一致。

第二阶段可将 ground plane 约束真正纳入深度求解中。也就是说，最终深度不能只取决于 depth map 和 2D 框尺寸，还应满足“放到地面上后仍然合理”这一条件。

#### 4.3.3 不再使用固定消失点

当前消失点更多是写死的比例值。第二阶段建议利用车道线方向来估计更真实的道路消失点或局部主方向，以减少因相机俯仰、裁剪范围、道路坡度造成的系统性偏差。

#### 4.3.4 depth 与 yaw 统一纳入“先验 + 观测”框架

这一阶段要避免把车道先验当作“额外补丁”，而是应建立更统一的思路：

1. 观测来自图像本身，如 bbox、mask、depth map。
2. 先验来自道路结构，如 lane direction、ground plane、车辆常见尺寸。
3. 最终的 depth 和 yaw 必须同时满足观测合理、先验合理、投影合理。

### 4.4 第二阶段预期效果

1. 车辆朝向会更加贴近车道方向，不再出现明显横偏。
2. 3D box 底面更贴地，尤其在应急车道边缘场景下更稳定。
3. 对固定机位和固定路段的泛化能力更强。

### 4.5 第二阶段风险

1. 车道分割若不稳定，会把错误的方向信息传递给 yaw。
2. 地面平面估计如果过于粗糙，可能反而带来额外偏差。
3. 在车辆换道、转向或遮挡严重时，车道先验不能完全替代视觉观测。

## 阶段三：考虑 keypoint / structured polygon / 轻量学习式 yaw 分支

### 5.1 阶段目标

当前系统最大的问题之一，是很多 3D 几何信息都来自启发式推断，而不是专门学习得到。第三阶段的目标是，在不引入超大模型和复杂训练链路的前提下，为系统增加一部分“可学习的几何感知能力”。

### 5.2 可选方向

#### 5.2.1 轻量学习式 yaw 分支

这是第三阶段最推荐的方向。

思路是：在当前检测结果基础上，为每辆车增加一个小型朝向估计分支，用于预测姿态角。

推荐采用“角度分类 + 小偏差回归”的方式，而不是直接回归一个连续角度。原因在于，角度具有周期性，直接回归容易在接近边界时不稳定，而“先判断属于哪个方向区间，再回归该区间内的小偏差”通常更稳。

这一方向的优点是：

1. 不需要替换现有检测主干。
2. 可以作为当前启发式 yaw 的补充或替代。
3. 对系统性偏航的修复最直接。

#### 5.2.2 keypoint-based 车辆结构估计

该方向的目标是让模型额外预测一些稳定的车辆结构点，例如：

1. 车头或车尾位置。
2. 底面角点。
3. 轮胎接地点近似位置。

一旦能稳定得到这些关键点，就可以更准确地恢复朝向、底面形状和 translation。

这一方向比单纯依赖 bbox 宽高比更接近真实几何结构，但其代价是需要更细致的标注或更复杂的监督设计。

#### 5.2.3 structured polygon / footprint estimation

你当前已经拥有车辆实例分割轮廓，因此这是一个很适合本项目的方向。与其从 mask 中后处理“猜”底面，不如显式地让系统学习或拟合一个更稳定的底面结构，例如四边形 footprint 或简化多边形。

这样做的好处是：

1. 比纯矩形 bbox 更符合车辆接地区域的真实形状。
2. 便于后续与车道区域做重叠判定。
3. 对违章识别任务本身更有帮助，因为你最终关心的是“车占了哪一块路面”。

### 5.3 第三阶段预期效果

1. yaw 估计更稳定、更少依赖手工规则。
2. 车辆底面恢复更加贴合真实占道区域。
3. 为后续向更完整的单目 3D 检测网络升级打下基础。

### 5.4 第三阶段风险

1. 可能需要新增数据标注或重新设计训练样本。
2. 若轻量分支训练数据不足，反而可能不如当前启发式稳定。
3. 工程复杂度和实验成本都会高于前两个阶段。

## 四、推荐执行顺序

从投入产出比来看，建议按照以下顺序实施：

1. 优先完成第一阶段，先解决 3D box 爆大和朝向系统性偏差。
2. 再进入第二阶段，把 lane/ground 先验纳入统一几何框架。
3. 最后再尝试第三阶段的小型学习式改造。

若只选择当前最值得先做的三项工作，建议是：

1. 优化深度采样与深度融合。
2. 去掉强硬的左右半边 yaw 符号规则。
3. 引入车道方向作为姿态先验。

## 五、核心概念解释

## 5.1 时序约束

时序约束是指：在视频连续帧中，同一辆车的几何属性不应该突然剧烈变化。

例如，一辆正常行驶的车在相邻两帧之间，不会突然从 30 米外跳到 10 米外，也不会在一帧中朝前、下一帧中突然横过来。因此，如果系统在某一帧估计出了明显异常的深度或朝向，就可以借助前后帧信息进行平滑和修正。

通俗理解，时序约束就是“看视频时，不只看这一张图，还参考前后几帧一起判断”。它特别适合违章识别这种工程任务，因为违章判定更看重稳定性，而不是单帧偶尔特别准。

## 5.2 translation

translation 可以理解为“物体在三维空间中的位置”。在相机坐标系中，通常用三个值来表示：

1. x：左右位置。
2. y：上下位置。
3. z：前后距离，也就是深度。

如果把 3D box 看成一个积木块，那么 translation 决定的是“把这个积木块放到空间中的哪里”。如果 translation 错了，即使箱子的大小和朝向都正确，最终画出来的位置也会不对。

## 5.3 yaw

yaw 是车辆在水平面上的朝向角。它描述的是车辆“向左偏了多少”或者“向右偏了多少”，不涉及上下俯仰。你可以把它理解成“从上往下看时，这辆车的朝向”。

## 5.4 local orientation 和 global yaw

这两个概念很容易混淆。

global yaw 指的是车辆在整个场景中的真实朝向；而 local orientation 更像是“从当前裁剪出来的车图里看，这辆车相对相机看起来偏了多少”。

之所以要分开，是因为同一辆真实朝前的车，出现在图像左边和图像右边时，它在局部 crop 中看起来是不一样的。很多单目 3D 检测方法不会直接从 crop 回归 global yaw，而是先回归 local orientation，再结合目标在图像中的位置恢复最终朝向。

## 5.5 prior

prior 可以理解为“先验知识”。也就是在真正开始估计之前，我们已经知道的一些规律。

例如：

1. 车辆通常位于地面上。
2. 高速道路上车辆大多沿车道方向行驶。
3. 小轿车的长宽高一般不会离谱。

这些信息都不是当前图片直接告诉我们的，而是我们提前知道的规律。把这些规律用于推理过程，就叫引入先验。

## 5.6 ground plane

ground plane 就是地面平面。对于道路场景，可以近似认为车辆都放在同一个平面上，因此合理的 3D box 应满足“底面贴地”。如果系统恢复出的 3D box 底面漂浮或插地，通常说明深度、朝向或位置估计出了问题。

## 5.7 重投影一致性

重投影一致性是单目 3D 检测中非常重要的概念。

意思是：当你恢复出一个 3D box 之后，把它重新投影回图像平面，它应该和原来的 2D 检测框、实例分割轮廓、接地区域等观测结果尽量吻合。

如果一个 3D 解投影回去之后明显对不上原图中的车辆轮廓，那么说明这个 3D 解本身不合理。重投影一致性本质上是一种“反向检查机制”。

## 六、后续工作建议

为了让后续实现更有条理，建议在正式改代码前先完成以下准备工作：

1. 梳理当前误差传播链，明确“深度偏近如何传导为 box 偏大”“yaw 偏差如何影响 footprint”。
2. 为第一阶段建立一组固定测试图像，便于对比优化前后效果。
3. 为第二阶段准备 lane direction 和 ground plane 的可视化验证手段，避免先验接入后难以判断是否有效。
4. 若考虑第三阶段，提前思考是否有条件构建轻量标注数据集，用于姿态角或关键点监督。

从工程实现角度看，最稳妥的路线不是一次性大改，而是每完成一个小优化，就用同一批样例图验证：

1. 3D box 是否更贴车体。
2. 底面是否更贴路面。
3. 朝向是否更符合车道方向。
4. 后续违章判定结果是否更稳定。

---

## 七、总结

对于 TrafficScan 当前的单目 3D 检测模块，最合理的优化路径并不是立刻更换成更大、更复杂的端到端 3D 网络，而是先基于已有系统做几何层面的修正，再逐步引入道路场景先验，最后才考虑轻量的学习式姿态与结构模块。

这种路线的优点在于：

1. 与现有工程兼容性高。
2. 可以较快看到效果改进。
3. 更符合当前项目的实际目标，即服务于违章识别，而不是单纯追求单目 3D 检测 benchmark 指标。

因此，推荐采用“阶段一修几何、阶段二加先验、阶段三补学习”的三阶段路线，作为后续单目 3D 检测优化的总体方案。
