# TrafficScan 单目 3D Box 核心算法说明

本文档面向计算机视觉和机器学习初学者，解释当前项目里单目 3D 目标检测模块到底在做什么、为什么这么做、以及公式的直观意义。

说明：

- 本文档描述的是项目当前实现对应的核心几何链路，而不是论文版完整 Deep3DBox 复现。
- 按当前需求，本文档先不讨论 `depth map` 融合，默认深度只使用 2D 框和车辆尺寸先验计算。

---

## 1. 这套算法到底在干什么

我们只有一张普通图片，却想恢复出：

- 车辆在三维空间中的位置
- 车辆朝向
- 车辆的 3D 长方体包围盒

这件事本质上不是“精确测量”，而是“带物理直觉的合理猜测”：

1. 用 2D 框大小猜车辆离相机多远
2. 用车框形状猜车辆朝向
3. 用固定车辆尺寸先验补足三维信息
4. 先在 3D 空间放一个标准车盒子，再投影回图像

可以把它理解成：

> 我们先猜这辆车有多远、头朝哪边，再拿一个标准鞋盒大小的 3D 盒子摆进去，最后看它投影回图像后是不是大致合理。

---

## 2. 坐标系怎么定义：XYZ 的正方向是什么

项目里使用的是**相机坐标系**，不是地图坐标系。

### 2.1 三维坐标方向

- `X`：向右
- `Y`：向下
- `Z`：向前，朝场景深处

也就是：

- 目标越靠图像右边，`X` 通常越大
- 目标越靠图像下边，`Y` 通常越大
- 目标越远离相机，`Z` 越大

### 2.1.1 相机坐标系 ASCII 示意图

```text
            场景深处
               Z+
               ^
               |
               |
               |
相机原点 O -----> X+
              \
               \
                v
                Y+（向下）
```

可以把相机想象成站在原点 `O`：

- 往右是 `X+`
- 往下是 `Y+`
- 往前方远处是 `Z+`

### 2.2 图像坐标方向

图像平面上的坐标记作 `(u, v)`：

- `u`：向右增大
- `v`：向下增大

所以这个项目里：

- 图像的“右”对应三维的 `X+`
- 图像的“下”对应三维的 `Y+`

这让投影公式更直接。

### 2.2.1 图像坐标 ASCII 示意图

```text
图像左上角 (0,0)
      +---------------------------> u+
      |
      |
      |
      v
      v+
```

### 2.3 相机内参的简化

项目没有使用严格标定相机参数，而是采用简化近似：

```text
f  = 1.2 * ImageWidth
cx = ImageWidth / 2
cy = ImageHeight / 2
```

其中：

- `f` 是焦距
- `(cx, cy)` 是主点，也就是相机光轴在图像上的中心

直观理解：

- `f` 越大，透视越“长焦”
- `(cx, cy)` 通常就是图像中心

## 3. 输入量和基础记号

设车辆 2D 检测框为：

```text
(x1, y1, x2, y2)
```

则：

### 3.1 框中心

```text
bx = (x1 + x2) / 2
by = (y1 + y2) / 2
```

### 3.2 框宽高

```text
bw = x2 - x1
bh = y2 - y1
```

### 3.3 车辆先验尺寸

项目为每种车辆类别给一个固定尺寸先验：

- `L`：车辆长度
- `Wv`：车辆宽度
- `Hv`：车辆高度

例如普通小汽车可近似理解为：

- `L ≈ 4.6 m`
- `Wv ≈ 1.9 m`
- `Hv ≈ 1.5 m`

这并不是每辆车的真实尺寸，而是“常见车辆平均尺寸”。

## 4. 3D Box 是如何从 2D 框恢复出来的

整体流程分三步：

1. 先估计车辆朝向 `yaw`
2. 再估计车辆深度 `z`
3. 用固定尺寸的 3D 盒子，旋转、平移，然后投影回图像

### 4.1 整体流程 ASCII 示意图

```text
2D 检测框 bbox
     |
     +--> 估计 yaw（朝向）
     |
     +--> 估计 depth（深度 z）
     |
     v
标准尺寸 3D 盒子
     |
     +--> 按 yaw 旋转
     |
     +--> 按 (Xc, Yc, Zc) 平移到相机坐标系
     |
     v
投影回图像平面
     |
     v
得到 3D box 的 2D 角点
```

## 5. 先在局部坐标系里造一个标准 3D 长方体

我们先不管车辆在图像中哪里，只先构造一个“标准车盒子”。

项目里使用的局部 3D 角点为：

```text
(+Wv/2, -Hv, +L/2)
(+Wv/2,  0,  +L/2)
(-Wv/2,  0,  +L/2)
(-Wv/2, -Hv, +L/2)
(+Wv/2, -Hv, -L/2)
(+Wv/2,  0,  -L/2)
(-Wv/2,  0,  -L/2)
(-Wv/2, -Hv, -L/2)
```

含义：

- `Y = 0` 是底面
- `Y = -Hv` 是车顶
- `X` 控制左右宽度
- `Z` 控制前后长度

直观理解：

> 就像先拿出一个标准鞋盒，准备把它摆进三维空间。

### 5.1 标准车盒子 ASCII 示意图

```text
            车顶（Y = -Hv）
             __________________
            /|               /|
           / |              / |
          /__|_____________/  |
          |  |             |  |
          |  |             |  |
          |  |             |  |
          |  /_____________|__/
          | /              | /
          |/_______________|/

            底面（Y = 0）

左右方向：X
前后方向：Z
高度方向：Y（向下为正，所以顶部是负值）
```

## 6. 车辆朝向 yaw 是如何计算的

`yaw` 可以理解为车辆绕竖直方向转了多少角度。

这个项目不是用神经网络直接回归 `yaw`，而是用几种几何线索来猜：

1. 2D 框宽高比
2. 车辆 mask 形状方向
3. 透视关系
4. 对小目标进行保守处理
5. 车道方向先验（如果车道检测可用）

---

## 7. 用 2D 框宽高比估计“侧视程度”

公式：

```text
ratio = bw / bh
apparent_width_m = ratio * Hv
side_bbox = clip((apparent_width_m - Wv) / (L - Wv), 0, 1)
```

下面逐行解释。

### 7.1 `ratio = bw / bh`

这是图像中车辆框的宽高比：

```text
ratio = 图像宽度 / 图像高度
```

直观理解：

- 如果框又高又窄，说明它更像正面或背面视角
- 如果框又扁又宽，说明它更像侧面视角

所以 `ratio` 本质上是一个“车辆在图像里有多横向展开”的信号。

### 7.2 `apparent_width_m = ratio * Hv`

这一项的作用是做一个粗略的“尺度还原”。

因为：

```text
ratio = 图像宽 / 图像高
```

如果我们已知真实车高大约是 `Hv` 米，那么：

```text
(图像宽 / 图像高) * 真实车高
```

就能得到一个近似的“横向可见真实尺寸”：

```text
apparent_width_m ≈ 车辆在当前视角下表现出来的等效横向宽度（米）
```

为什么这样近似是合理的？

因为在同一深度下：

- 图像尺寸与真实尺寸成比例
- 宽度和高度同时受距离缩放
- 宽高相除以后，距离的影响大致被抵消

### 7.3 `side_bbox = clip((apparent_width_m - Wv) / (L - Wv), 0, 1)`

这一步是在问：

> 当前看到的横向宽度，更像真实车宽，还是更像真实车长？

两个极端：

- 如果车辆几乎正对你：
  - 横向可见尺寸更接近 `Wv`
- 如果车辆几乎侧对你：
  - 横向可见尺寸更接近 `L`

所以用下面的归一化：

```text
side_bbox = 0   -> 更像正视/背视
side_bbox = 1   -> 更像纯侧视
```

完整公式：

```text
side_bbox = clip((apparent_width_m - Wv) / (L - Wv), 0, 1)
```

其中 `clip(a, 0, 1)` 的意思是：

- 小于 0 就当成 0
- 大于 1 就当成 1

### 7.4 一句最直观的话

这个公式其实就在做一件事：

> 看这辆车在图像里显得有多“横着展开”，然后把它映射成“更像正视还是更像侧视”。

### 7.5 `side_bbox` 的直观示意

```text
正视/背视                    斜视                       侧视

  ____                    ________                 _____________
 |    |                  /       /|               |             |
 |    |                 /______/ /                |             |
 |____|                 |      |/                 |_____________|

框更高更窄              框逐渐变宽                框更扁更宽
ratio 较小              ratio 中等                ratio 较大
side_bbox -> 0          side_bbox -> 中间值       side_bbox -> 1
```

## 8. 什么是车辆 mask，什么是主轴方向

项目里车辆分割模型会输出每辆车的 `mask`，你可以理解成：

> 图像里哪些像素属于这辆车

在代码里常用的是车辆轮廓多边形，也就是一堆二维点：

```text
[(x1, y1), (x2, y2), (x3, y3), ...]
```

这不是 3D 点云，而是 **2D mask 轮廓点集**。

### 8.1 什么是主轴方向

想象把这些点洒在纸上：

- 如果车看起来更横着，这堆点会沿横向拉得更长
- 如果车看起来更竖着，这堆点会沿纵向拉得更长

“主轴方向”就是：

> 这堆点延展得最厉害的那个方向

就像一根铅笔：

- 长方向是主轴
- 短方向是次轴

### 8.2 数学上怎么找主轴方向

项目中采用点集协方差矩阵的主特征向量来近似：

1. 将点集平移到中心附近：

```text
centered = pts - mean(pts)
```

2. 计算协方差矩阵：

```text
cov = centered^T * centered
```

3. 求特征分解，取最大特征值对应的特征向量：

```text
principal = 主特征向量
```

这个 `principal` 就是 mask 的主轴方向。

---

## 9. `side_mask = horizontal / (horizontal + vertical)` 是什么意思

设主轴方向向量为：

```text
principal = [px, py]
```

其中：

- `px` 表示它在 `x` 方向上的分量
- `py` 表示它在 `y` 方向上的分量

### 9.1 什么叫 x/y 分量绝对值

因为我们这里只关心“更水平还是更垂直”，不关心它到底朝左还是朝右，所以取绝对值：

```text
horizontal = |px|
vertical   = |py|
```

举例：

```text
principal = [0.98, 0.20]
```

表示主轴几乎水平，因为 x 分量很大。

```text
principal = [0.10, 0.99]
```

表示主轴几乎竖直，因为 y 分量很大。

### 9.2 `side_mask` 的意义

公式：

```text
side_mask = horizontal / (horizontal + vertical)
```

含义：

- 如果主轴很水平：
  - `horizontal` 大
  - `vertical` 小
  - `side_mask` 接近 1
- 如果主轴很竖直：
  - `horizontal` 小
  - `vertical` 大
  - `side_mask` 接近 0

所以：

```text
side_mask ≈ 1  -> 更像侧视
side_mask ≈ 0  -> 更像正视/背视
```

### 9.3 一句直观总结

`side_bbox` 看的是“框有多扁宽”，  
`side_mask` 看的是“分割轮廓整体是横着拉长还是竖着拉长”。

### 9.4 mask 主轴方向 ASCII 示意

#### 情况 A：主轴更水平，像侧视

```text
      ********************
   **************************
  ****************************
   **************************
      ********************

主轴方向大致是：
---------------------------->

horizontal 大，vertical 小
side_mask 接近 1
```

#### 情况 B：主轴更竖直，像正视/背视

```text
          ******
        **********
        **********
        **********
        **********
          ******

主轴方向大致是：
           |
           |
           v

horizontal 小，vertical 大
side_mask 接近 0
```

## 10. 如何把 `side_bbox` 和 `side_mask` 融合

如果有车辆 mask，则项目里采用轻量融合：

```text
side = 0.9 * side_bbox + 0.1 * side_mask
```

如果没有 mask，则：

```text
side = side_bbox
```

这表示当前实现里仍然更信任 2D 框几何，mask 只是提供一个辅助修正。

---

## 11. 从“侧视程度”变成 yaw 几何幅值

公式：

```text
yaw_geom = side * (pi / 2)
```

含义：

- `side = 0` 时，`yaw_geom = 0°`
- `side = 1` 时，`yaw_geom = 90°`

也就是说：

> 我们把“更像正视还是更像侧视”这个比例，映射成一个角度大小。

---

## 12. 什么叫透视项

只看 bbox 和 mask 还不够，因为高速公路图像有明显透视结构：

- 越远的车辆会朝消失点汇聚
- 图像左边和右边的车，从视觉上会显得朝不同方向倾斜

所以项目加入了一个“透视项”。

### 12.1 消失点

项目里近似定义消失点为：

```text
vp_x = ImageWidth / 2
vp_y = 0.3 * ImageHeight
```

可以把它理解成：

> 远方道路线条在图像里大致汇聚到的位置

### 12.2 透视项公式

```text
dx = |vp_x - bx|
dy = max(0.22 * ImageHeight, |by - vp_y|)
yaw_persp = atan2(dx, dy)
```

### 12.3 透视项的直观意义

这里在做的事情是：

- `dx`：车辆离消失点左右偏了多少
- `dy`：车辆离消失点上下偏了多少

如果车辆离消失点左右偏得越大，说明它视觉上越像“斜着朝远方延伸”。

于是用：

```text
yaw_persp = atan2(dx, dy)
```

把这种透视倾斜感转换成一个角度。

### 12.4 为什么 `dy` 要设下限

如果一辆车很靠近地平线，`dy` 可能非常小。  
这样 `atan2(dx, dy)` 会突然变得很大，导致 `yaw` 爆掉。

所以项目设置：

```text
dy >= 0.22 * ImageHeight
```

意思是：

> 别让远处靠近地平线的小车，因为像素波动而被误判成非常大的朝向角。

---

## 13. 什么叫“小目标降权”

公式：

```text
area_ratio = (bw * bh) / (ImageWidth * ImageHeight)
damp = 0.55 + 0.45 * clip((area_ratio - 0.0015) / 0.012, 0, 1)
yaw_abs = yaw_abs * damp
```

### 13.1 它要解决什么问题

远处小车的问题特别多：

- 分辨率低
- bbox 不稳定
- mask 不稳定
- 宽高比非常容易抖动

如果我们对远处小车的朝向过度自信，就很容易出错。

所以这里做了一件很朴素的事：

> 对特别小的目标，少信一点。

### 13.3 小目标降权 ASCII 示意

```text
大目标（近）                     小目标（远）

      ____________                 ____
     |            |               |    |
     |   vehicle  |               |car |
     |____________|               |____|

信息更丰富                       信息更少、噪声更大
更敢估大角度                     更保守，yaw 往小压
```

### 13.2 怎么理解这个公式

先算车辆框面积占整张图的比例：

```text
area_ratio = 车辆框面积 / 图像面积
```

然后得到一个缩放系数 `damp`：

- 目标越小，`damp` 越接近 0.55
- 目标越大，`damp` 越接近 1.0

最后：

```text
yaw_abs = yaw_abs * damp
```

意思是把小目标的 yaw 拉小一些，使估计更保守。

一句最直观的话：

> 这辆车太小了，我看不清，所以不要太武断地说它转了很大角度。

---

## 14. 最终 yaw 如何得到

### 14.1 几何项 + 透视项融合

```text
yaw_abs = 0.7 * yaw_geom + 0.3 * yaw_persp
```

### 14.2 小目标降权后

```text
yaw_abs = yaw_abs * damp
```

补充说明：当前实现里，`yaw` 在经过上面的几何融合和小目标降权之后，
如果当前帧已经检测到车道区域，并且能从车道边界估计出车辆所在位置附近的车道走向，
还会再引入一个 `lane_prior` 对 `yaw_abs` 做一次软融合。

```text
lane_prior = {
    direction: [dx, dy],
    confidence: c
}

lane_prior_yaw_abs = atan2(|dx|, |dy|)
lane_weight = clip(0.18 + 0.62 * confidence, 0, 0.80)
yaw_abs = (1 - lane_weight) * yaw_abs + lane_weight * lane_prior_yaw_abs
```

它的含义是：

- `direction` 表示车道在图像中的主方向
- `confidence` 表示这个车道方向估计有多可信
- 车道方向不是完全替代 bbox、mask 和透视线索，而是在可信时把 `yaw_abs` 往“顺着车道走向”的方向轻轻拉过去

### 14.3 yaw 的符号

项目里还需要决定“往左转还是往右转”的方向符号。当前实现采用一个视觉启发式：

```text
if bx < cx:
    sign = +1
else:
    sign = -1
```

补充说明：如果有车道先验，当前实现里还会额外参考车道方向来修正符号。

```text
sign_score = 0.65 * visual_sign + 0.95 * confidence * lane_prior_sign
```

其中：

- `visual_sign` 是上面这个“相对图像中心”的视觉符号
- `lane_prior_sign` 来自车道方向横向分量的符号

如果 `sign_score` 明显偏正或偏负，就会把最终符号往对应方向拉；
如果车道先验置信度足够高，那么项目会直接采用车道先验的符号。

于是最终：

```text
yaw = sign * yaw_abs
```

这不是严格物理求解，而是基于道路透视场景的一种经验规则。

## 15. 深度 z 如何计算（先禁用 depth map）

在本说明里，默认不使用深度图，只使用几何先验深度。

思路很简单：

- 近的车看起来大
- 远的车看起来小

所以只要知道：

- 图像里的框高
- 图像里的框宽
- 真实车辆大概有多高、多宽

就可以反推大致距离。

### 15.1 yaw 会影响“看起来有多宽”

公式：

```text
apparent_width = |Wv * cos(yaw)| + |L * sin(yaw)|
```

直观理解：

- 正视时，横向看到的主要是车宽 `Wv`
- 侧视时，横向看到的主要是车长 `L`
- 斜着时，横向可见尺寸介于两者之间

这相当于在说：

> 你在图里看到的横向宽度，并不是固定的，它会随着朝向变化。

### 15.2 用车高反推深度

```text
z_h = f * Hv / bh
```

含义：

- 真实高度 `Hv` 已知
- 图像高度 `bh` 已知
- 根据针孔相机模型可反推距离

这表达的是一个很朴素的事实：

> 车在图里越高，通常离你越近；车在图里越矮，通常离你越远。

### 15.3 用车宽反推深度

```text
z_w = f * apparent_width / bw
```

含义类似：

- 真实横向可见尺寸已知
- 图像宽度已知
- 就能反推距离

### 15.4 融合两种深度

```text
z = 0.65 * z_h + 0.35 * z_w
z = clip(z, MIN_DEPTH, MAX_DEPTH)
```

为什么高度权重大一点？

因为当前实现里：

- 车高通常更稳定
- 横向宽度更容易受 yaw 误差影响

所以项目更信任由高度推出来的深度。

## 16. 车辆 3D 中心坐标如何计算

当 `yaw` 和深度 `z` 有了以后，车辆 3D 中心位置按下面公式计算：

```text
Xc = (bx - cx) * z / f
Yc = (y2 - cy) * z / f
Zc = z
```

解释如下：

- `Zc = z`：这辆车离相机多远
- `Xc`：根据车框中心在图像中的左右偏移，还原它在三维中的左右位置
- `Yc`：根据 2D 框底边在图像中的上下位置，还原它在三维中的上下位置

为什么 `Yc` 用的是 `y2` 而不是 `by`？

因为项目希望把 3D 盒子的**底面中心**大致锚定到车辆接地点附近，而 `y2` 更接近轮胎接地位置。

### 16.1 3D 中心恢复 ASCII 示意

```text
图像平面

      x1          bx            x2
       |-----------|-------------|
       +-------------------------+
       |                         |
       |         vehicle         |
       |                         |
       +-------------------------+ <- y2（框底边）

恢复思路：

bx - cx   -> 决定三维里的左右位置 Xc
y2 - cy   -> 决定三维里的上下位置 Yc
z         -> 决定三维里的前后位置 Zc
```

## 17. 3D 盒子如何旋转和平移

### 17.1 绕 Y 轴旋转

```text
R(yaw) =
[ cos(yaw)   0   sin(yaw) ]
[    0       1      0     ]
[ -sin(yaw)  0   cos(yaw) ]
```

### 17.2 平移到相机坐标系

设局部坐标中的点为 `P_local`，则相机坐标中的点为：

```text
P_cam = R(yaw) * P_local + [Xc, Yc, Zc]^T
```

这一步的物理意义就是：

1. 先把标准车盒子转到合适方向
2. 再把它放到三维空间中的正确位置

## 18. 如何把 3D 点投影回 2D 图像

对于相机坐标中的 3D 点 `(X, Y, Z)`，项目使用标准针孔投影：

```text
u = f * X / Z + cx
v = f * Y / Z + cy
```

直观理解：

- 东西越远（`Z` 越大），投影越小
- 东西越靠右（`X` 越大），图像横坐标 `u` 越大
- 东西越靠下（`Y` 越大），图像纵坐标 `v` 越大

于是 3D 盒子的 8 个角点都会被投到图像上，形成最终绘制出来的 3D box。

```text
       /       /|                       /--------/|
      /_______/ |                      |        | |
      |       | |        ----->        |        |/
      |       | /                      /--------/
      |_______|/

真实 3D 盒子                         图像里的 2D 线框
```

## 19. 整个核心算法总公式

### 19.1 输入

```text
bbox = (x1, y1, x2, y2)
bw = x2 - x1
bh = y2 - y1
bx = (x1 + x2) / 2
by = (y1 + y2) / 2
```

### 19.2 相机参数

```text
f  = 1.2 * ImageWidth
cx = ImageWidth / 2
cy = ImageHeight / 2
```

### 19.3 bbox 侧视程度

```text
ratio = bw / bh
apparent_width_m = ratio * Hv
side_bbox = clip((apparent_width_m - Wv) / (L - Wv), 0, 1)
```

### 19.4 mask 侧视程度

```text
principal = mask 主轴方向
horizontal = |principal_x|
vertical   = |principal_y|
side_mask = horizontal / (horizontal + vertical)
```

### 19.5 侧视程度融合

```text
side = 0.9 * side_bbox + 0.1 * side_mask
```

### 19.6 几何 yaw

```text
yaw_geom = side * (pi / 2)
```

### 19.7 透视 yaw

```text
dx = |vp_x - bx|
dy = max(0.22 * ImageHeight, |by - vp_y|)
yaw_persp = atan2(dx, dy)
```

### 19.8 融合后的 yaw 幅值

```text
yaw_abs = 0.7 * yaw_geom + 0.3 * yaw_persp
```

### 19.9 小目标降权

```text
area_ratio = (bw * bh) / (ImageWidth * ImageHeight)
damp = 0.55 + 0.45 * clip((area_ratio - 0.0015) / 0.012, 0, 1)
yaw_abs = yaw_abs * damp
```

补充：如果车道检测可用，还会进一步做一次车道先验修正：

```text
lane_prior_yaw_abs = atan2(|dx|, |dy|)
lane_weight = clip(0.18 + 0.62 * confidence, 0, 0.80)
yaw_abs = (1 - lane_weight) * yaw_abs + lane_weight * lane_prior_yaw_abs
```

### 19.10 yaw 符号

```text
if bx < cx:
    sign = +1
else:
    sign = -1

yaw = sign * yaw_abs
```

补充：如果车道先验置信度较高，符号还会被车道方向覆盖或强烈拉动。

### 19.11 朝向影响下的可见横向宽度

```text
apparent_width = |Wv * cos(yaw)| + |L * sin(yaw)|
```

### 19.12 深度（禁用 depth map）

```text
z_h = f * Hv / bh
z_w = f * apparent_width / bw
z = clip(0.65 * z_h + 0.35 * z_w, MIN_DEPTH, MAX_DEPTH)
```

### 19.13 车辆 3D 中心

```text
Xc = (bx - cx) * z / f
Yc = (y2 - cy) * z / f
Zc = z
```

### 19.14 3D 旋转和平移

```text
P_cam = R(yaw) * P_local + [Xc, Yc, Zc]^T
```

### 19.15 投影回图像

```text
u = f * X / Z + cx
v = f * Y / Z + cy
```

## 20. 最后用一句最直观的话总结

这套算法并不是在“神奇地恢复真实三维世界”，而是在做一件非常朴素的事情：

> 用 2D 框大小猜距离，用形状猜朝向，再把一个标准车盒子放进三维空间，看它投影回图片后是否合理。

公式看起来很多，但本质上都在表达三个常识：

1. 近的东西看起来更大，远的东西看起来更小
2. 横向展开得越厉害，越像侧视
3. 三维物体放进空间后，要遵守透视投影规律
