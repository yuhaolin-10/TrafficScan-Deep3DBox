# TrafficScan 单目3D检测阶段二规划文档

## 一、文档目标

本文档用于规划 TrafficScan 单目 3D 检测优化的第二阶段工作。第二阶段的核心任务不是继续单纯修补几何链路，而是把项目本身已经具备的道路场景信息正式接入 3D 几何恢复过程，使 depth 和 yaw 不再只依赖 2D 检测框、实例分割轮廓和 depth map，而是开始真正利用：

1. 车道方向先验
2. 地面平面约束
3. 高速道路固定机位的场景稳定性

第一阶段的目标是“让框别太离谱”，第二阶段的目标则是“让框开始真正符合道路场景”。

---

## 二、第二阶段的核心问题

在第一阶段之后，系统会比原来更稳，但仍然存在几个本质限制：

1. yaw 仍然主要来自目标外观启发式，而不是来自道路结构约束。
2. depth 仍然主要由深度图和几何先验融合得到，地面约束尚未真正参与优化。
3. 消失点与道路方向仍然更多依赖粗略估计，未充分利用 lane mask。
4. 当前几何恢复仍以“单车”为中心，没有充分利用“这是一条高速路，车辆大多沿车道前进”的事实。

因此，第二阶段必须把“场景先验”从辅助信息，升级为几何恢复中的正式约束。

---

## 三、第二阶段总体目标

第二阶段建议围绕以下 4 个目标展开：

1. 从 lane mask 中提取稳定的局部车道方向。
2. 用 lane direction 约束 yaw，而不是仅靠 bbox 和 mask 主轴猜朝向。
3. 用 ground plane 约束车辆底面位置和 depth。
4. 建立“观测 + 先验 + 重投影”统一框架，让 depth 和 yaw 同时满足图像观测和道路结构约束。

---

## 四、模块级规划

## 4.1 车道方向提取模块

### 目标

从现有 `lane_mask` 或 `lane_polygons` 中提取可用于几何约束的方向信息。

### 建议新增或改造位置

1. [`src/core/lane_segmenter.py`](d:\college\GraduationProject\testProject\TrafficScan_lab_copy\src\core\lane_segmenter.py)
2. 可新增一个轻量工具模块，例如：
   - `src/core/lane_geometry.py`
   - 或 `src/core/road_geometry.py`

### 需要输出的信息

建议至少输出以下一种或多种信息：

1. 全局车道主方向
2. 每辆车所在位置附近的局部车道切线方向
3. 估计的道路消失点

### 实现思路

可以按由简到难的顺序做：

1. 从 lane polygon 的主轴估计整条车道的主方向
2. 在车辆底面 anchor 附近，提取局部 lane edge 的切线方向
3. 多条 lane edge 延长相交，估计更稳定的 vanishing point

### 风险

1. lane mask 如果边界锯齿过大，方向估计可能抖动
2. 仅有应急车道区域时，局部切线提取可能不如完整车道线稳定

---

## 4.2 ground plane 建模模块

### 目标

将“车辆底面必须落在路面上”这一事实正式纳入 3D box 恢复过程。

### 建议改造位置

1. [`src/core/vehicle_detector_3d.py`](d:\college\GraduationProject\testProject\TrafficScan_lab_copy\src\core\vehicle_detector_3d.py)
2. 可拆出：
   - `src/core/ground_geometry.py`

### 当前问题

当前项目里虽然已有 `GroundPlaneEstimator`，但它更多还是启发式占位实现，并没有真正把 ground plane 作为 depth 优化中的强约束。

### 第二阶段建议

将 ground plane 用在两个地方：

1. depth 约束
   车辆底面点应位于地面附近，若某个 depth 使得车辆底面明显漂浮或插地，则应降低其可信度

2. footprint 约束
   底面多边形不仅要和 2D mask 对得上，还要和地面方向一致

### 推荐输出

1. 路面法向量的近似表达
2. 地平线/消失线位置
3. 某个 3D box 是否满足“贴地约束”的评分

---

## 4.3 yaw 估计模块升级

### 目标

让 yaw 从“外观启发式”为主，升级为“车道方向先验 + 图像观测残差”为主。

### 建议改造位置

1. [`src/core/vehicle_detector_3d.py`](d:\college\GraduationProject\testProject\TrafficScan_lab_copy\src\core\vehicle_detector_3d.py)

### 第二阶段建议思路

将 yaw 分成两部分：

1. lane-based base yaw
   由车道方向给出车辆主朝向基线

2. visual residual yaw
   由 bbox、mask、透视关系估计一个小偏差

最终：

`final yaw = lane base yaw + residual yaw`

### 这样做的好处

1. 高速场景下，朝向不会再大幅偏离道路方向
2. 对远车更稳定，因为远车外观信息弱，但车道方向仍然可靠
3. 可以减少“因为目标在图像左右位置不同而符号乱跳”的问题

---

## 4.4 depth 融合模块升级

### 目标

让 depth 不再只是“深度图 + 尺寸先验”的融合，而是加入 ground/lane 场景约束。

### 建议改造位置

1. [`src/core/vehicle_detector_3d.py`](d:\college\GraduationProject\testProject\TrafficScan_lab_copy\src\core\vehicle_detector_3d.py)

### 第二阶段建议思路

第二阶段可以将 depth 候选来源扩展为：

1. 几何先验深度
2. depth map 深度
3. ground-constrained 深度
4. lane-consistent 深度

再基于以下指标做融合：

1. 重投影误差
2. 贴地误差
3. 车道方向一致性
4. 目标外观质量

最终目标不是求某一个“理论上最正确的深度”，而是求一个在当前场景中最自洽的深度。

---

## 五、第二阶段推荐实施顺序

第二阶段建议按以下顺序推进，而不是并行全改。

### 步骤 1：做 lane direction 可视化实验

先不要急着改 yaw 逻辑。先验证：

1. lane mask 是否足够稳定
2. lane polygon 是否能提取出合理的主方向
3. 局部切线方向是否能跟车辆实际行驶方向对上

验收重点：

1. 方向箭头是否稳定
2. 左右车道位置不同的车辆是否仍能得到一致的道路主方向

### 步骤 2：把 lane direction 接到 yaw 估计中

在保留现有视觉 yaw 的前提下，先做小规模融合，不要一步替换。

建议先做：

1. 用 lane direction 作为 yaw 候选之一
2. 与现有视觉 yaw 一起做重投影评分比较

验收重点：

1. 朝向整体是否更顺着车道
2. 是否减少横偏和符号乱跳

### 步骤 3：实现贴地评分

先不要把 ground plane 直接变成复杂几何模型，先实现一个“贴地合理性评分”。

例如检查：

1. 底面是否漂浮
2. 底面是否过度插地
3. 底面左右倾斜是否异常

验收重点：

1. 一些明显漂的框，评分应更差
2. 合理贴地的框，评分应更好

### 步骤 4：将贴地评分并入 depth 决策

在已有 depth 融合基础上，加入 ground 相关约束。

验收重点：

1. 远车深度是否更稳定
2. 近车是否不再容易向外爆框
3. footprint 是否更贴路面

### 步骤 5：统一成“观测 + 先验 + 重投影”框架

这一阶段最后再整理结构，把 lane prior 和 ground prior 正式纳入统一评分或优化逻辑中，避免代码继续长成多个补丁叠加的样子。

---

## 六、第二阶段验收标准

当第二阶段完成时，建议至少达到以下效果：

1. 大多数车辆的 yaw 与车道方向基本一致
2. 同一车道上的车辆朝向更统一，不再出现明显左右乱跳
3. footprint 更贴路面，悬浮感减少
4. depth 在固定机位场景下更稳定
5. 远距离车辆的 3D box 比第一阶段进一步收敛

---

## 七、第二阶段交付物建议

建议形成以下交付物：

1. 一版 lane direction 可视化结果图
2. 一版贴地评分或 ground 约束可视化结果
3. 一版对比图：
   - 第一阶段结果
   - 第二阶段结果
4. 一份阶段总结，说明：
   - 哪些误差因 lane prior 得到明显改善
   - 哪些误差仍需要第三阶段学习式模块解决

---

## 八、与第三阶段的衔接

第二阶段的意义不仅是让当前系统更稳，还在于为第三阶段打基础。

如果第二阶段顺利完成，第三阶段就不必再从零学习一个完全自由的 yaw，而可以只学习：

1. lane-based yaw 的小残差
2. 更稳定的底面结构
3. 更适合违章判定的 footprint

换句话说，第二阶段做得越扎实，第三阶段需要学习的自由度越小，训练难度和数据需求也会越低。

---

## 九、总结

第二阶段的本质，是把 TrafficScan 从“通用图像几何恢复”推向“面向高速道路场景的结构化几何恢复”。

第一阶段解决的是“几何别太离谱”，第二阶段解决的则是“几何要符合道路逻辑”。因此，阶段二最关键的不是继续微调单个公式，而是把车道方向和地面平面正式纳入 3D box 恢复过程，让系统真正利用已有的 lane/ground 信息。
