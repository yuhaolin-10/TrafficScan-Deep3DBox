M1.2 目标（在不接入模型/数据库前提下把交互体验补齐）

- 让用户在“导入 → 选择 → 预览 → 运行(伪) → 看结果(伪) → 反馈日志”的链路上，感受到完整产品交互，而不是占位 UI
- 保持当前 M1.1 的稳定启动与可维护结构：仍然不引入后端推理，不做真实渲染，只做“体验与组件能力”补齐
M1.2 功能拆分（建议按面板逐个落地，方便 review）

- 1) Viewer（预览体验）
  
  - 视图模式：Fit to Window（默认）/ 1:1 / 自定义缩放比例（比如 25%~400%）
  - 基础交互：滚轮缩放、双击切换 Fit↔1:1、拖拽平移（可选，但很推荐）
  - 图层开关 UI（先不接渲染）：Lane Mask / Footprint / 3D Box / Labels / Highlight
  - 状态提示：无图时显示英文占位提示；加载失败显示英文错误提示（不抛异常到控制台）
- 2) Workspace（任务队列体验）
  
  - 任务状态占位：PENDING / RUNNING / DONE / FAILED（先由 UI 模拟更新）
  - 批量操作入口：Clear / Remove Selected / Run Selected / Run All（Run 两个你已经有按钮了）
  - 多选策略：支持多选后，“Run Selected”针对多选；单选时保持现有逻辑
  - 去重与过滤延续 M1.1（不改变你的导入链路，只是把“状态”展示出来）
- 3) Violations Table（表格交互补齐，mock 数据）
  
  - 行选中联动：点击行 → Viewer 高亮“选中车辆”（先做 UI 高亮占位，不需要真框）
  - Confirmed 勾选交互：支持勾选/取消勾选；先不落库
  - mock 数据来源：按“Run Selected/All”生成固定结构的假结果填充表格（确保 UI 逻辑真实可跑）
- 4) Logs（可用性增强）
  
  - 右键菜单：Copy Selected / Copy All / Clear
  - 批量运行时进度占位：例如 “3/20 finished, elapsed …”（先用定时器模拟）
M1.2 关键“状态模型”（不接后端也要先把数据流定下来）

- WorkspaceItem（概念上）： path + status + (optional) result
- UI 运行行为（仍然是 UI-only）：
  - Run Selected/All 触发后：把目标条目的 status 置为 RUNNING
  - 用短延时模拟处理完成：置为 DONE，并生成 mock violations 结果
  - 失败路径也要有：比如随机或对特定文件名触发 FAILED，用于验证 UI 分支完整
M1.2 验收标准（你一眼能判断是否达标）

- 导入图片后：Workspace 显示任务且有状态字段（默认 PENDING）
- Viewer：Fit/1:1/缩放可切换，窗口 resize 时不会“被吞图”或无限放大
- Run Selected/All：会更新状态（RUNNING→DONE/FAILED），并填充右侧表格（mock）
- 表格交互：选中行有高亮，Confirmed 可勾选，Viewer 有“高亮占位反馈”
- 全程 UI 文案为英文，且窗口可稳定启动运行
