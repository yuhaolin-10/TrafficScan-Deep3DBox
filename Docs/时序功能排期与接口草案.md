# 时序功能排期与接口草案

## 目标

这一轮先不把三个功能一次性全部做完，而是把它们正式纳入新工程的主线，并把接口、数据结构和存储提前定下来。

涉及需求：

1. 手拉违停区域
2. 连续帧车流量统计
3. 车牌识别

---

### 1. 旧 3D 检测器已下线

  - 现在只保留兼容代理，统一转到 Deep3DBox 检测器

  - `src/core/vehicle_detector_deep3dbox.py`

## 后续实施顺序

## 第一阶段：手拉违停区域

目标：

- 在 GUI 中给 `ImageViewer` 增加 polygon 绘制模式
- 支持保存当前固定机位的人工违停区域
- 支持在后续连续帧处理中优先使用人工区域

建议改动：

- `src/gui/image_viewer.py`
- `src/gui/main_window.py`
- `src/services/scene_profile.py`

## 第二阶段：连续帧车流量统计

目标：

- 引入 `track_id`
- 对同一车辆的轨迹做连续帧关联
- 用 `TrafficCounter` 做计数线穿越统计

建议改动：

- `src/core/traffic_counter.py`
- `src/services/pipeline.py`
- 新增视频/序列处理 worker

## 第三阶段：车牌识别

目标：

- 接入车牌检测 + OCR
- 对同一 `track_id` 在多帧上的车牌候选做融合
- 将结果写入 `plate_reads`

建议改动：

- `src/core/plate_recognizer.py`
- `src/services/database_manager.py`
- GUI 事件详情面板

## 结论

这三个需求现在已经不只是“想法”，而是已经有了：

- 项目排期
- 数据结构
- 数据库预留
- 算法骨架

接下来最合理的顺序仍然是：

1. 先做手拉违停区域
2. 再做连续帧车流量统计
3. 最后接车牌识别
