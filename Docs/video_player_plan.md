# 视频播放器开发清单

## 目标

在当前 `TrafficScan_Deep3DBox_UI` 里先做一个轻量级视频播放器效果，满足以下最小可用需求：

- 选中视频后可以看到当前帧
- 有时间轴
- 可以播放 / 暂停
- 可以上一帧 / 下一帧
- 处理完成后优先逐帧预览“处理后视频结果”
- 保持现有 `ImageViewer` 的缩放、ROI、预览风格，不强行切成另一套播放器控件

---

## 为什么先这样做

当前项目已经有：

- `ImageViewer`
- `ViewerPanel`
- `VideoReader`
- 处理后视频输出能力

所以最省风险的路线不是先重做成完整多媒体播放器，而是：

1. 继续用 `ImageViewer` 显示单帧
2. 用 `QTimer + slider + buttons` 驱动帧切换
3. 视频只负责“按帧取图”，界面继续沿用现有预览体系

这样好处是：

- ROI 逻辑不用重写
- 以后点击目标、看检测框、看处理结果更自然
- 代码改动范围可控

---

## 可借用的开源 / 官方方案

### 方案 A：Qt 官方播放器思路

可借用内容：

- `QMediaPlayer`
- `QVideoWidget`
- `positionChanged / durationChanged / setPosition`
- 官方播放器示例的控件布局方式

适用：

- 以后如果要做真正流畅的视频播放器
- 原视频播放体验优先

本阶段不作为主实现，只借布局和交互思路。

### 方案 B：当前项目自建帧播放器

核心思路：

- 保留 `ImageViewer`
- 新增时间轴和播放按钮
- 用 OpenCV 按帧读取视频
- 用 `QTimer` 推进播放

适用：

- 当前项目最合适
- 最容易与 ROI / 检测结果叠加结合

### 方案 C：PyAV

适合后续情况：

- 需要更稳定的 seek
- 某些编码格式在 OpenCV 下不稳定
- 需要更精确的帧级控制

当前阶段先不引入，避免依赖变重。

## 本轮开发范围

### 1. 文档与设计

- 新建本文档
- 明确采用“帧播放器”方案
- 明确后续可升级为 Qt Multimedia / PyAV

### 2. 底层视频会话层

新增一个可复用的视频会话对象，用于：

- 打开视频一次，不要每切一帧都重新打开
- 支持顺序读帧
- 支持 seek 到某一帧
- 支持关闭释放资源

建议位置：

- `src/services/video_reader.py`

### 3. ViewerPanel 增加视频控件

新增：

- 播放 / 暂停按钮
- 上一帧 / 下一帧按钮
- 时间轴 slider
- 当前帧 / 总帧数 / 时间标签
- 当前预览来源标签：原视频 / 处理结果

要求：

- 只有选中视频时显示
- 选中图片时隐藏
- 不影响现有 ROI 工具栏

### 4. MainWindow 集成播放状态

新增能力：

- 维护当前视频播放状态
- 维护当前视频帧索引
- 维护 `QTimer`
- 处理 slider seek
- 处理播放推进
- 处理切换文件时释放旧 session

### 5. 处理结果优先预览

规则：

- 视频未处理：播放原视频
- 视频处理完成：优先播放处理后视频
- 视频处理中：仍播放原视频，避免读正在写入的输出文件

### 6. 最小验证

- 编译检查
- 用本地小视频验证：
  - 播放
  - 暂停
  - seek
  - 上一帧 / 下一帧
- 验证处理完成后会切到处理结果视频

---

## 本轮不做的内容

- 真正的音视频播放器
- 音频
- 倍速播放
- 拖拽区间循环
- 帧级目标点击检测联动
- 原视频 / 结果视频手动切换按钮
- 时间轴事件标记点

---

## 预计开发顺序

1. 先做视频会话层
2. 再做 `ViewerPanel` 视频控件 UI
3. 再接 `MainWindow` 状态流转
4. 最后做 smoke test 和细节修正
