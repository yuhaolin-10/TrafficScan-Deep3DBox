# Thesis Workspace

This folder is a dedicated workspace for the graduation thesis of the project.

## Files

- `outline.md`: thesis outline aligned with the current system architecture
- `draft_thesis.md`: a first full draft based on the real modules in this repo
- `materials_checklist.md`: figures, tables, metrics, and screenshots still worth补充
- `figures/`: recommended location for thesis figures
- `references/`: recommended location for reference exports or notes
- `materials/`: recommended location for experiment records, screenshots, and tables

## Suggested workflow

1. Revise the title and chapter names to match your school template.
2. Replace metric placeholders in `draft_thesis.md` with your real experiment results.
3. Export CNKI references into `references/` and add them to the reference section.
4. Put GUI screenshots, flowcharts, and result images into `figures/`.
5. When the structure is stable, split the draft into chapter files or migrate it into Word.

## Current writing baseline

The current draft is grounded in the actual project modules, including:

- YOLO-based vehicle detection
- lane segmentation
- monocular 3D bounding box estimation based on Deep3DBox
- region rule engine for violation reasoning
- object tracking, plate recognition, GUI, and database persistence

It is intentionally written as a strong first draft rather than a final school-formatted submission.
